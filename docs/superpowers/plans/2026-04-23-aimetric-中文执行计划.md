# AIMetric 中文执行计划

## 1. 项目目标

AIMetric 目标是同构复现文章《AI出码率70%+的背后：高德团队如何实现AI研发效率的量化与优化》中的 AI 研发效能量化平台。

复现重点不是简单做一个 dashboard，而是复现文章里的核心链路：

```text
AI 编码工具 / MCP 工具
  -> 采集端
  -> 采集网关
  -> 事件存储
  -> Git / AI 归因
  -> 指标计算
  -> 个人 / 团队看板
```

文章同构分层：

- `采集平台层`：Cursor、CLI、IDE 插件等入口
- `数据采集层`：MCP 工具、采集 SDK、采集网关
- `平台能力层`：事件存储、归因、指标计算、规则体系、知识库查询
- `指标展示层`：个人出码视图、团队出码视图、会话分析、出码分析

## 8. 下一步建议

建议优先级如下：

1. 建立企业级指标语义层
2. 员工端轻量化安装与多 AI 工具接入
3. 建设面向提效管理者和技术管理者的多维分析工作台
4. 接入研发交付系统、质量系统和 ROI 模型
5. 接入 Agent / RAG 智能分析层

最推荐下一步：

```text
企业级指标语义层
```

原因：

- 当前主链路、增强采集、证据链、Dashboard、鉴权、审计、可观测和运维手册已经跑通
- 下一阶段目标已经从文章复现升级为企业级 AI 研发效能平台
- 企业级平台首先需要统一指标口径，否则多工具采集、Dashboard 和 Agent 分析都会各自定义指标，后期很难治理
- 下一阶段完整计划见：`docs/superpowers/plans/2026-04-24-enterprise-ai-effectiveness-platform-plan.md`
