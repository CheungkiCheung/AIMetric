# AIMetric AI 工具度量驾驶舱 Walkthrough

本文档用于试点演示和联调，帮助提效管理者按一条固定路径理解 AIMetric 的价值。演示主线不是“页面有哪些”，而是“企业投入的 AI 工具是否被使用、是否形成有效交付、是否带来质量代价、下一步应该采取什么管理动作”。

## 1. 演示前准备

先确保：

- `collector-gateway` 已启动
- `metric-platform` 已启动
- `dashboard` 已启动
- 已导入一批演示数据

导入演示数据：

```bash
corepack pnpm demo:seed
```

只看导入计划，不真正写入：

```bash
corepack pnpm demo:seed:dry
```

先做联调检查：

```bash
corepack pnpm demo:check
```

如果检查失败，优先确认：

- `collector-gateway` 是否运行在 `127.0.0.1:3000`
- `metric-platform` 是否运行在 `127.0.0.1:3001`
- 当前 `projectKey` 是否与导入数据一致

## 2. 推荐演示顺序

建议按下面顺序讲：

1. `AI 工具度量驾驶舱`
   先说明 AIMetric 的第一入口是面向提效管理者的工具度量驾驶舱，核心关注 Cursor、MCP、CLI Agent、Codex、Claude Code、VS Code / JetBrains 等工具的接入状态、渗透、产出、交付和风险。
2. `工具资产与接入状态`
   展示已接入、接入就绪、观测中工具，以及工具类别、健康状态和当前聚焦工具。
3. `趋势与场景渗透`
   展示活跃、会话、AI 触达需求、AI 触达 PR、Tab 接受等趋势，让管理者先判断 adoption 是否真实发生。
4. `工具影响漏斗`
   从接入、活跃、会话、PR 触达、有效产出一路看转化，避免只看单点会话数。
5. `效率与质量成本`
   同时展示 Lead Time、PR 周转、Review、CI、缺陷、发布失败、事故关联，避免“局部更快、整体更差”。
6. `管理动作板`
   把结论落到扩大推广、补充培训、修复采集、质量治理、暂停推广或深入分析。
7. `组织治理概览`
   先说明这不是单一看板，而是有组织、团队、项目、成员治理边界的平台。
8. `权限治理配置`
   说明技术管理者和平台管理员看到的范围可以不同。
9. `采集健康运营`
   说明平台先保证“采得到、采得稳”，再谈指标。
10. `个人出码视图 / 团队出码视图`
   展示 AI 使用渗透和基础有效产出。
11. `需求交付概览 / PR 交付概览`
   展示 AI 是否真正进入需求与交付流程。
12. `CI 质量概览 / 发布质量概览 / 事故风险概览 / 缺陷风险概览`
   展示速度和质量是否平衡。
13. `缺陷归因分析`
   重点讲：
   - `AI 参与需求缺陷率`
   - `AI 触达 PR 逃逸缺陷率`
   - `发布失败关联缺陷数`
   - `事故关联缺陷数`
14. `统一指标计算管线`
   说明平台现在不是散点指标，而是统一语义层。

## 3. 建议重点讲的业务问题

演示时不要只讲“系统做了什么”，更建议围绕问题讲：

- AI 有没有真正被用起来
- 哪些 AI 工具值得继续推广，哪些只是接入了但没有形成产出
- AI 是否进入需求和 PR，而不只是停留在聊天
- 速度提升有没有带来 Review、CI、缺陷、事故风险
- 哪些风险可以归因到需求、PR、发布或事故链路
- 哪些指标已经足够做试点，哪些经营指标还不该过早做重
- 下一步管理动作是扩大、培训、修复采集、质量治理还是暂停推广

## 4. 本轮最适合演示的指标

建议重点挑这些：

- `AI 出码率`
- `AI 触达需求占比`
- `AI 触达 PR 占比`
- `lead_time_ai_vs_non_ai`
- `critical_requirement_cycle_time`
- `pr_cycle_time`
- `review_rejection_rate`
- `ci_pass_rate`
- `change_failure_rate`
- `rollback_rate`
- `defect_rate`
- `escaped_defect_rate`
- `AI 参与需求缺陷率`
- `AI 触达 PR 逃逸缺陷率`

## 5. 演示结束时可以给出的结论

当前这版最适合给出的结论是：

```text
这套系统已经可以支撑企业 AI 研发提效试点，
重点价值不只是“看 AI 出码率”，
而是把 AI 工具资产、员工接入、采集健康、需求、PR、CI、发布、事故、缺陷和管理动作连成一条产品闭环。
```

不建议在这个阶段过度承诺：

- 精准 ROI
- 精确人力成本节省
- 全组织统一经营结论

更好的说法是：

```text
先把趋势、对比、归因和治理做扎实，再进入 Agent / RAG 智能分析阶段。
```
